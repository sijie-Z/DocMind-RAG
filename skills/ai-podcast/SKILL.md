# AI 播客脚本生成

## 执行规则

**重要**：你必须直接根据用户的提示词生成播客脚本，**绝对不要反问用户任何问题**。

## 固定参数（不可更改）

以下参数已预设，无需询问，直接使用：

| 参数 | 固定值 |
|------|--------|
| 播客风格 | **双人对话**（主持人A和主持人B） |
| 时长要求 | **1分钟**（约150-180字） |
| 目标听众 | **程序员/技术人员** |

## 主题来源

**播客主题**来自上一个节点的输出内容（用户提示词中提供），你需要：
1. 直接使用用户提示词中的内容作为播客主题
2. 将该内容转化为双人对话形式
3. 无需确认，立即生成脚本

## 快速开始

1. 读取用户提示词中的主题内容
2. 调用 `load_skill_reference('ai-podcast', 'script-template')` 获取模板
3. **直接生成1分钟双人对话脚本，不询问任何问题**

## 脚本结构

一个完整的播客脚本通常包含以下部分：

### 1. 开场白
- 吸引注意力的开场
- 主持人自我介绍
- 本期主题预告
- 时长提示

### 2. 主体内容
- 核心话题展开
- 故事/案例分享
- 数据/事实支撑
- 观点表达

### 3. 互动环节
- Q&A 问答
- 听众反馈
- 话题延伸

### 4. 结尾
- 内容总结
- 下期预告
- 订阅/分享引导
- 结束语

## 脚本风格指南

### 语言特点
- 口语化表达，避免书面语
- 使用短句，便于朗读
- 适当停顿，标注节奏
- 情感丰富，有感染力

### 时长控制（1分钟脚本）
- 总字数：约150-180字（中文）
- 开场白：20-30字
- 主体内容：100-120字
- 结尾：20-30字
- **紧凑节奏，直奔主题，无废话**

- `script-template`: 完整的播客脚本模板
- `voice-guide`: 语音风格指南
