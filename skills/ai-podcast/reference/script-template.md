# 1分钟双人对话播客模板

**固定格式**：双人对话（主持人A + 主持人B）
**目标听众**：程序员/技术人员
**总时长**：约1分钟（150-180字）

---

## 标准模板

```
【开场】(约20字)

A：大家好，欢迎来到程序员播客！
B：今天我们来聊聊【主题】。

【主体对话】(约120字)

A：说到【主题】，我觉得最关键的是【核心观点】。
B：没错！而且【补充观点】，这对我们程序员特别重要。
A：对，【具体例子或应用场景】。
B：【进一步解释或反面例子】。
A：【总结或实际建议】。

【结尾】(约20字)

B：时间关系今天就到这，下次见！
A：下期再见！
```

---

## 示例脚本

**主题：Git Commit 最佳实践**

```
A：大家好，欢迎来到程序员播客！
B：今天我们来聊聊 Git Commit 的最佳实践。

A：说到这个，我觉得最重要的是写好 commit message。
B：没错！一个好的 commit message 应该说清楚"做了什么"和"为什么做"。
A：对，比如用 "feat: 添加用户登录功能" 就比 "更新代码" 强一百倍。
B：哈哈是的，那种只写 "fix bug" 的提交，一周后自己都看不懂。
A：所以记住，type + scope + subject，清清楚楚。

B：时间关系今天就到这，下次见！
A：下期再见！
```

## 朗读提示标记

| 标记 | 含义 |
|------|------|
| [停顿] | 短暂停顿 |
| [笑声] | 自然笑声 |
| [强调] | 重读标记 |

---

## 注意事项

1. **对话要自然**：像两个程序员在茶水间聊天
2. **有信息量**：1分钟内要传递有价值的技术内容
3. **节奏紧凑**：不要客套话，直奔主题
4. **互相呼应**：B要接A的话，形成对话感
