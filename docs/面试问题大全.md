### 1. Transformer 是什么？

**核心回答：**

Transformer 是 2017 年 Google 在论文《Attention Is All You Need》中提出的神经网络架构，是现代大语言模型（LLM）的基础。

**关键组件：**

```
┌─────────────────────────────────────────────────────────────┐
│                    Transformer 架构                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  输入 Embedding + 位置编码 (Positional Encoding)             │
│                    ↓                                         │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              Encoder (编码器) - N 层                 │    │
│  │  ┌─────────────────────────────────────────────┐    │    │
│  │  │     Multi-Head Self-Attention (多头自注意力) │    │    │
│  │  │     Q (Query) × K (Key) × V (Value)         │    │    │
│  │  │     Attention(Q,K,V) = softmax(QK^T/√d)V    │    │    │
│  │  └─────────────────────────────────────────────┘    │    │
│  │                      ↓                               │    │
│  │  ┌─────────────────────────────────────────────┐    │    │
│  │  │     Feed-Forward Network (前馈网络)          │    │    │
│  │  │     FFN(x) = max(0, xW1 + b1)W2 + b2        │    │    │
│  │  └─────────────────────────────────────────────┘    │    │
│  └─────────────────────────────────────────────────────┘    │
│                    ↓                                         │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              Decoder (解码器) - N 层                 │    │
│  │  Masked Self-Attention + Cross-Attention + FFN      │    │
│  └─────────────────────────────────────────────────────┘    │
│                    ↓                                         │
│              输出概率分布 (softmax)                          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**为什么 Transformer 如此重要？**

1. **并行计算**：不像 RNN 需要顺序处理，Transformer 可以并行处理整个序列
2. **长距离依赖**：Self-Attention 可以直接建模任意位置之间的关系
3. **可扩展性**：可以堆叠更多层、增加参数量（GPT-4 有万亿参数）
4. **通用性**：同一个架构可以处理文本、图像、音频、视频

**面试追问：**

- Q: Self-Attention 的时间复杂度是多少？如何优化？
- A: O(n²)，n 是序列长度。优化方法包括稀疏注意力、线性注意力、Flash Attention 等。

- Q: GPT 和 BERT 的区别是什么？
- A: GPT 是 Decoder-only 架构，自回归生成；BERT 是 Encoder-only 架构，双向理解。GPT 适合生成任务，BERT 适合理解任务。

### 2. 什么是 LLM（大语言模型）？

**核心回答：**

LLM 是基于 Transformer 架构、在海量文本数据上训练的大型神经网络模型，具备语言理解、生成、推理等能力。

**主流 LLM 对比：**

| 模型 | 公司 | 参数量 | 特点 |
|------|------|--------|------|
| GPT-4 | OpenAI | ~1.7T (推测) | 综合能力最强 |
| Claude | Anthropic | 未知 | 安全性、长文本处理 |
| DeepSeek | 深度求索 | 67B/236B | 国产开源，推理能力强 |
| 通义千问 | 阿里 | 7B-72B | 多模态、开源 |
| GLM | 智谱 | 6B-130B | 中英双语、开源 |

**LLM 的核心能力：**

```
┌────────────────────────────────────────────────────────────┐
│                    LLM 能力金字塔                           │
├────────────────────────────────────────────────────────────┤
│                                                             │
│                    ┌─────────┐                              │
│                    │  推理   │  ← 逻辑推理、数学、编程      │
│                  ┌─┴─────────┴─┐                            │
│                  │   理解      │  ← 阅读理解、意图识别      │
│                ┌─┴─────────────┴─┐                          │
│                │     生成        │  ← 文本生成、翻译、摘要  │
│              ┌─┴─────────────────┴─┐                        │
│              │       知识          │  ← 世界知识、常识      │
│            ┌─┴─────────────────────┴─┐                      │
│            │        语言             │  ← 语法、词汇、表达  │
│            └─────────────────────────┘                      │
│                                                             │
└────────────────────────────────────────────────────────────┘
```

**LLM 的局限性：**

1. **幻觉问题**：生成看似合理但实际错误的内容
2. **知识截止**：训练数据有截止日期，不知道最新信息
3. **上下文长度限制**：虽然现在有 128K-1M token 的模型，但仍有成本问题
4. **缺乏实时性**：无法访问实时数据
5. **数学推理不稳定**：复杂计算容易出错

### 3. RAG 是什么？

**核心回答：**

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将外部知识库与大语言模型结合的技术架构，通过检索相关文档来增强 LLM 的回答质量和准确性。

**为什么需要 RAG？**

```
┌─────────────────────────────────────────────────────────────┐
│                    LLM 的痛点 vs RAG 解决                    │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ❌ 知识截止 → ✅ 实时检索最新知识                           │
│  ❌ 幻觉问题 → ✅ 基于真实文档回答                           │
│  ❌ 领域知识不足 → ✅ 接入企业知识库                         │
│  ❌ 数据隐私 → ✅ 私有化部署，数据不出域                     │
│  ❌ 无法溯源 → ✅ 提供引用来源                               │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**RAG 架构详解：**

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          RAG 系统架构                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │                      离线索引阶段                                  │   │
│  │                                                                   │   │
│  │   文档 ─→ 解析(PDF/Word) ─→ 分块(Chunking) ─→ 向量化(Embedding)   │   │
│  │                                                        ↓          │   │
│  │                                              向量数据库(Vector DB)  │   │
│  │                                              - Elasticsearch       │   │
│  │                                              - Milvus              │   │
│  │                                              - Pinecone            │   │
│  └──────────────────────────────────────────────────────────────────┘   │
│                                                                          │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │                      在线查询阶段                                  │   │
│  │                                                                   │   │
│  │   用户问题                                                        │   │
│  │       ↓                                                           │   │
│  │   Query改写 ─→ 向量化 ─→ 相似度搜索 ─→ 重排序(Rerank)             │   │
│  │                                          ↓                        │   │
│  │                                   相关文档片段                     │   │
│  │                                          ↓                        │   │
│  │                              Prompt 构建（问题 + 文档）            │   │
│  │                                          ↓                        │   │
│  │                                    LLM 生成回答                    │   │
│  │                                          ↓                        │   │
│  │                              返回答案 + 引用来源                   │   │
│  └──────────────────────────────────────────────────────────────────┘   │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

**核心概念详解：**

#### 3.1 向量嵌入（Embedding）

将文本转换为高维向量表示，语义相似的文本在向量空间中距离更近。

```python
# 示例：文本转向量
from zhipuai import ZhipuAI
client = ZhipuAI()

response = client.embeddings.create(
    model="embedding-3",  # 2048 维向量
    input="什么是人工智能？"
)
# 输出: [0.123, -0.456, 0.789, ...] 共2048个浮点数
```

**主流 Embedding 模型：**

| 模型 | 维度 | 特点 |
|------|------|------|
| OpenAI text-embedding-3-small | 1536 | 性价比高 |
| OpenAI text-embedding-3-large | 3072 | 效果最好 |
| ZhipuAI embedding-3 | 2048 | 中文效果好 |
| BGE-large-zh | 1024 | 开源中文模型 |
| m3e-base | 768 | 轻量级中文模型 |

#### 3.2 文本分块（Chunking）

将长文档切分成适合检索的小块，是 RAG 质量的关键。

**分块策略对比：**

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 固定长度 | 按字符数切分 | 简单快速 | 可能切断语义 |
| 句子分割 | 按标点切分 | 保持句子完整 | 块大小不均匀 |
| 语义分块 | 基于语义相似度切分 | 保持语义完整 | 计算成本高 |
| 递归分块 | 按层级结构切分 | 保持文档结构 | 需要了解文档结构 |
| 父子分块 | 小块检索，大块喂给LLM | 检索精准+上下文丰富 | 实现复杂 |

```python
# 语义分块示例
from semantic_chunker import SemanticChunker

chunker = SemanticChunker(
    embedding_model="embedding-3",
    similarity_threshold=0.5,  # 相似度阈值
    min_chunk_size=100,        # 最小块大小
    max_chunk_size=500         # 最大块大小
)

chunks = chunker.chunk(document_text)
```

#### 3.3 相似度搜索

在向量数据库中找到与查询最相似的文档块。

**常用相似度计算方法：**

```python
# 余弦相似度（最常用）
def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# 欧氏距离
def euclidean_distance(a, b):
    return np.linalg.norm(a - b)

# 点积（用于已归一化的向量）
def dot_product(a, b):
    return np.dot(a, b)
```

#### 3.4 混合检索（Hybrid Search）

结合关键词检索和向量检索，提高召回率。

```
┌─────────────────────────────────────────────────────────────┐
│                      混合检索流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  用户查询 ──┬──→ 向量检索 ──→ Top-K 向量结果                │
│             │                     ↓                         │
│             │              RRF 融合排序                      │
│             │                     ↑                         │
│             └──→ BM25检索 ──→ Top-K 关键词结果              │
│                                                              │
│  RRF 公式: score(d) = Σ 1/(k + rank(d))                    │
│  其中 k 通常取 60                                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

#### 3.5 重排序（Rerank）

对检索结果进行二次排序，提高相关性。

```python
# Cross-Encoder Rerank 示例
from zhipuai import ZhipuAI

client = ZhipuAI()
response = client.rerank(
    model="bge-reranker-v2-m3",
    query="什么是机器学习？",
    documents=retrieved_docs,  # 初步检索结果
    top_n=5                    # 返回前5个最相关的
)
```

**RAG 面试高频问题：**

1. **Q: RAG 和微调（Fine-tuning）怎么选？**

   | 维度 | RAG | Fine-tuning |
   |------|-----|-------------|
   | 知识更新 | 实时更新 | 需要重新训练 |
   | 成本 | 低 | 高 |
   | 适用场景 | 需要实时数据、企业知识库 | 需要特定领域风格、格式 |
   | 可解释性 | 高（有引用来源） | 低 |
   | 推荐 | 知识密集型任务 | 风格/格式敏感任务 |

2. **Q: 如何评估 RAG 系统效果？**
   - **检索指标**：召回率(Recall)、精确率(Precision)、MRR、NDCG
   - **生成指标**：BLEU、ROUGE、BERTScore
   - **端到端指标**：答案正确性、相关性、忠实度、答案完整性
   - **RAGAS 框架**：专门用于 RAG 评估

3. **Q: 如何优化 RAG 的召回效果？**
   - 多路召回（向量 + 关键词 + 知识图谱）
   - Query 改写和扩展
   - HyDE（假设文档嵌入）
   - 向量数据库索引优化（IVF、HNSW）
   - 重排序模型

### 4. Agent 是什么？

**核心回答：**

Agent（智能体）是能够感知环境、做出决策、执行动作以实现目标的自主系统。在 AI 领域，Agent 通常指基于 LLM 的智能代理，能够自主规划和执行复杂任务。

**Agent vs 传统软件：**

```
┌─────────────────────────────────────────────────────────────┐
│                    传统软件 vs Agent                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  传统软件：                                                  │
│  ┌───────────┐     ┌───────────┐     ┌───────────┐         │
│  │  输入 A   │ ─→  │  固定逻辑 │ ─→  │  输出 B   │         │
│  └───────────┘     └───────────┘     └───────────┘         │
│  （确定性、可预测、需要人工编码每个场景）                     │
│                                                              │
│  Agent：                                                     │
│  ┌───────────┐     ┌───────────┐     ┌───────────┐         │
│  │  目标/任务 │ ─→  │ 自主规划  │ ─→  │ 动态执行  │         │
│  └───────────┘     │  + 决策   │     │  + 反馈   │         │
│                    └───────────┘     └───────────┘         │
│  （自主性、适应性、能够处理未见过的情况）                     │

└─────────────────────────────────────────────────────────────┘
```

### 5. Agent 记忆系统

**核心回答：**

Agent 记忆系统是让 AI 智能体能够存储、检索和利用历史信息的关键组件，直接决定了 Agent 的长期表现和个性化能力。

**记忆系统分类：**

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Agent 记忆系统架构                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ┌───────────────────────────────────────────────────────────────────┐  │
│  │                      短期记忆 (Short-term Memory)                  │  │
│  │                                                                    │  │
│  │  - 存储内容: 当前对话上下文                                        │  │
│  │  - 存储方式: LLM Context Window (内存)                            │  │
│  │  - 生命周期: 会话级别                                              │  │
│  │  - 典型实现: ConversationBufferMemory, 滑动窗口                   │  │
│  │  - 挑战: 上下文长度限制 (4K-128K tokens)                          │  │
│  │                                                                    │  │
│  │  优化策略:                                                         │  │
│  │  - 滑动窗口: 只保留最近N轮对话                                    │  │
│  │  - 摘要压缩: 定期总结历史对话                                     │  │
│  │  - Token裁剪: 根据重要性裁剪                                      │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                                                          │
│  ┌───────────────────────────────────────────────────────────────────┐  │
```

```python
# 综合记忆检索示例
class MemoryRetriever:
    def retrieve(self, query: str, user_id: str):
        # 1. 向量相似度检索
        semantic_results = self.vector_db.search(
            query_embedding=self.embed(query),
            filter={"user_id": user_id},
            top_k=10
        )
        
        # 2. 时间衰减因子
        current_time = time.time()
        for result in semantic_results:
            age_hours = (current_time - result.timestamp) / 3600
            result.score *= math.exp(-age_hours / 168)  # 一周半衰期
        
        # 3. 重要性权重
        for result in semantic_results:
            result.score *= result.importance_weight
        
        # 4. 重排序
        return sorted(semantic_results, key=lambda x: x.score, reverse=True)[:5]
```

### 2. RAG 核心流程详解

```python
# 本系统的 RAG 完整流程

class RAGService:
    async def search(self, query: str, organization_id: int) -> List[Document]:
        """
        完整的 RAG 检索流程
        """
        # 1. 查询预处理
        processed_query = await self._preprocess_query(query)
        
        # 2. 查询意图分类
        intent = await self._classify_intent(processed_query)
        
        # 3. 查询改写（可选多查询扩展）
        expanded_queries = await self._expand_query(processed_query, intent)
        
        # 4. 混合检索
        results = []
        for q in expanded_queries:
            # 4.1 向量检索
            vector_results = await self._vector_search(q, organization_id)
            # 4.2 关键词检索
            keyword_results = await self._keyword_search(q, organization_id)
            # 4.3 RRF 融合
            merged = self._rrf_fusion(vector_results, keyword_results)
            results.extend(merged)
        
        # 5. MMR 多样性选择
        diverse_results = self._mmr_selection(results, top_k=10)
        
        # 6. 重排序
        reranked = await self._rerank(processed_query, diverse_results)
        
        # 7. 过滤和阈值处理
        filtered = self._filter_by_threshold(reranked, threshold=0.5)
        
        return filtered[:5]
    
    async def generate_answer(
        self, 
        query: str, 
        documents: List[Document],
        conversation_history: List[Message]
    ) -> AsyncGenerator[str, None]:
        """
        流式生成回答
        """
        # 1. 构建 Prompt
        prompt = self._build_rag_prompt(query, documents, conversation_history)
        
        # 2. 调用 LLM 流式生成
        async for chunk in self.llm_client.stream_generate(prompt):
            yield chunk
```

### 3. Agent 工作流引擎

本系统采用 LangGraph 实现状态图工作流：

```python
# 工作流状态定义
class WorkflowState(TypedDict):
    input: Dict[str, Any]        # 用户输入
    messages: List[Any]          # 对话消息
    current_node: Optional[str]  # 当前执行节点
    node_outputs: Dict[str, Any] # 各节点输出
    context: Dict[str, Any]      # 共享上下文
    errors: List[str]            # 错误列表

# 节点执行器基类
class NodeExecutor:
    async def execute(self, state: WorkflowState) -> Dict[str, Any]:
        raise NotImplementedError

# LLM 节点执行器
class LLMNodeExecutor(NodeExecutor):
    async def execute(self, state: WorkflowState) -> Dict[str, Any]:
        # 1. 获取 LLM 实例
        llm = self.get_llm(model_type=self.node.type)
        
        # 2. 构建消息
        messages = [
            SystemMessage(content=self.node.data["systemPrompt"]),
            *state["messages"]
        ]
        
        # 3. 调用 LLM
        response = await llm.ainvoke(messages)
        
        # 4. 返回状态更新
        return {
            "messages": [response],
            "node_outputs": {self.node.id: {"content": response.content}}
        }

# 工作流引擎
class WorkflowEngine:
    def build_graph(self) -> StateGraph:
        workflow = StateGraph(WorkflowState)
        
        # 添加节点
        for node in self.nodes:
            workflow.add_node(node.id, self._create_node_func(node))
        
        # 设置入口点
        workflow.set_entry_point(self._find_entry_node())
        
        # 添加边（支持条件边）
        for edge in self.edges:
            if self._is_conditional(edge):
                workflow.add_conditional_edges(
                    edge.source, 
                    self._create_router(edge)
                )
            else:
                workflow.add_edge(edge.source, edge.target)
        
        return workflow
```

### 2. Agent 相关

**Q1: Agent 如何处理工具调用失败？**

```
策略:
1. 重试机制: 指数退避重试
2. 降级方案: 调用备用工具
3. 错误反馈: 将错误信息喂回 LLM 重新规划
4. 人工介入: 设置失败阈值，触发人工审核

示例代码:
async def execute_with_retry(tool, params, max_retries=3):
    for i in range(max_retries):
        try:
            result = await tool.execute(params)
            return result
        except Exception as e:
            if i == max_retries - 1:
                # 反思并尝试替代方案
                alternative = await llm.plan_alternative(str(e))
                return await execute_with_retry(alternative.tool, alternative.params)
            await asyncio.sleep(2 ** i)
```

**Q2: 如何防止 Agent 无限循环？**

```
策略:
1. 最大步数限制: 设置 max_iterations
2. 状态检测: 检测重复状态
3. 进度评估: 每步评估是否接近目标
4. 超时机制: 设置总执行时间上限

示例:
class AgentLoopDetector:
    def __init__(self, max_same_state=3):
        self.state_history = []
        self.max_same_state = max_same_state
    
    def check_loop(self, current_state: dict) -> bool:
        state_hash = hash(frozenset(current_state.items()))
        self.state_history.append(state_hash)
        
        # 检测重复状态
        if self.state_history.count(state_hash) >= self.max_same_state:
            return True
        return False
```

```
协调机制:
1. 中央调度: Supervisor Agent 分配任务

选择依据:
- 任务类型: 独立任务用中央调度，协作任务用共享黑板

```

### 1. Agent 记忆系统设计

**Q: 如何设计一个生产级的 Agent 记忆系统？**

```python
# 生产级 Agent 记忆系统架构
class ProductionMemorySystem:
    """
    分层记忆系统
    """
    def __init__(self):
        # 短期记忆: Redis
        self.short_term = RedisMemory()
        # 长期记忆: 向量数据库
        self.long_term = VectorMemory()
        # 工作记忆: 状态存储
        self.working = StateMemory()
        # 反思记忆: 结构化存储
        self.reflective = ReflectiveMemory()
    
    async def remember(self, agent_id: str, event: Event):
        """记忆存储"""
        # 1. 存入短期记忆
        await self.short_term.add(agent_id, event)
        
        # 2. 重要性评估
        importance = await self._evaluate_importance(event)
        
        # 3. 高重要性事件转长期记忆
        if importance > 0.7:
            await self.long_term.add(agent_id, event, importance)
        
        # 4. 定期反思（触发条件：事件数量或时间）
        if await self._should_reflect(agent_id):
            insights = await self._reflect(agent_id)
            await self.reflective.add(agent_id, insights)
    
    async def recall(self, agent_id: str, query: str, context: dict):
        """记忆检索"""
        results = []
        
        # 1. 从工作记忆获取当前任务状态
        working_state = await self.working.get(agent_id)
        results.append(working_state)
        
        # 2. 从短期记忆获取最近对话
        recent = await self.short_term.get_recent(agent_id, limit=10)
        results.extend(recent)
        
        # 3. 从长期记忆语义检索
        relevant = await self.long_term.search(
            agent_id, 
            query, 
            time_decay=True,  # 时间衰减
            importance_weight=True  # 重要性权重
        )
        results.extend(relevant)
        
        # 4. 从反思记忆获取相关洞察
        insights = await self.reflective.search(agent_id, query)
        results.extend(insights)
        
        return self._deduplicate_and_rank(results)
    
    async def _reflect(self, agent_id: str) -> List[Insight]:
        """反思过程：从事件中提取高层洞察"""
        recent_events = await self.short_term.get_recent(agent_id, limit=100)
        
        prompt = f"""
        分析以下事件，提取关键洞察和模式：
        {json.dumps(recent_events, ensure_ascii=False)}
        
        请输出：
        1. 发现的模式
        2. 成功经验
        3. 失败教训
        4. 改进建议
        """
        
        reflection = await self.llm.generate(prompt)
        return self._parse_insights(reflection)
```

### 2. 知识图谱与 GraphRAG

**Q: 如何从非结构化文档构建知识图谱？**

```python
# 知识图谱构建流程
class KnowledgeGraphBuilder:
    def __init__(self):
        self.llm = get_llm()
        self.graph_db = Neo4jClient()
    
    async def build_from_documents(self, documents: List[Document]):
        """从文档构建知识图谱"""
        for doc in documents:
            # 1. 文档分块
            chunks = self.chunk_document(doc)
            
            for chunk in chunks:
                # 2. 实体抽取
                entities = await self.extract_entities(chunk)
                
                # 3. 关系抽取
                relations = await self.extract_relations(chunk, entities)
                
                # 4. 实体链接（消歧）
                linked_entities = await self.link_entities(entities)
                
                # 5. 存入图数据库
                await self.save_to_graph(linked_entities, relations)
    
    async def extract_entities(self, text: str) -> List[Entity]:
        """实体抽取"""
        prompt = f"""
        从以下文本中抽取实体，类型包括：人物、组织、地点、事件、概念
        
        文本：{text}
        
        输出格式：
        [
            {{"name": "实体名", "type": "类型", "description": "描述"}},
            ...
        ]
        """
        result = await self.llm.generate(prompt)
        return json.loads(result)
    
    async def extract_relations(self, text: str, entities: List[Entity]):
        """关系抽取"""
        prompt = f"""
        从以下文本和实体中抽取关系
        
        文本：{text}
        实体：{entities}
        
        输出格式：
        [
            {{"head": "实体1", "relation": "关系", "tail": "实体2"}},
            ...
        ]
        """
        result = await self.llm.generate(prompt)
        return json.loads(result)
    
    async def graph_rag_search(self, query: str) -> List[dict]:
        """GraphRAG 检索"""
        # 1. 识别查询中的实体
        query_entities = await self.extract_entities(query)
        
        # 2. 图谱查询（多跳）
        cypher = f"""
        MATCH (e:Entity)-[r*1..3]-(related:Entity)
        WHERE e.name IN {query_entities}
        RETURN e, r, related
        LIMIT 50
        """
        subgraph = await self.graph_db.execute(cypher)
        
        # 3. 转换为文本
        context = self.graph_to_text(subgraph)
        
        # 4. 结合向量检索
        vector_results = await self.vector_search(query)
        
        # 5. 融合返回
        return self.merge_results(context, vector_results)
```

### 3. 文稿撰写 Agent 设计

**Q: 如何设计一个报告生成智能体？**

```python
# 报告生成 Agent 架构
class ReportGenerationAgent:
    def __init__(self):
        self.planner = ReportPlanner()
        self.researcher = ResearchAgent()
        self.writer = WritingAgent()
        self.reviewer = ReviewAgent()
        
    async def generate_report(self, topic: str, requirements: dict):
        """生成完整报告"""
        
        # 1. 规划阶段：生成报告大纲
        outline = await self.planner.plan(topic, requirements)
        
        # 2. 研究阶段：收集素材
        materials = {}
        for section in outline.sections:
            materials[section.id] = await self.researcher.research(
                section.topic,
                sources=requirements.get('sources', ['web', 'knowledge_base'])
            )
        
        # 3. 写作阶段：逐节撰写
        report = Report()
        for section in outline.sections:
            draft = await self.writer.write_section(
                section=section,
                materials=materials[section.id],
                style=requirements.get('style', 'academic')
            )
            report.add_section(draft)
        
        # 4. 审核阶段：质量检查
        review_result = await self.reviewer.review(report)
        
        # 5. 修订阶段
        if review_result.needs_revision:
            for issue in review_result.issues:
                await self.revise(report, issue)
        
        # 6. 格式化输出
        return await self.format_output(report, requirements.get('format', 'markdown'))
    
    async def revise(self, report: Report, issue: Issue):
        """修订报告"""
        # 定位问题段落
        section = report.get_section(issue.section_id)
        
        # 重新撰写
        revised = await self.writer.revise(
            section=section,
            issue=issue,
            strategy=issue.fix_strategy
        )
        
        report.update_section(issue.section_id, revised)


class ReportPlanner:
    """报告规划器"""
    async def plan(self, topic: str, requirements: dict) -> Outline:
        prompt = f"""
        主题：{topic}
        要求：
        - 字数：{requirements.get('word_count', 5000)}
        - 类型：{requirements.get('type', 'research')}
        - 风格：{requirements.get('style', 'academic')}
        
        请生成报告大纲，包括：
        1. 标题
        2. 摘要
        3. 章节（每章包含小节）
        4. 各章节预估字数
        5. 关键论点
        
        输出 JSON 格式。
        """
        result = await self.llm.generate(prompt)
        return Outline.parse(result)


class ResearchAgent:
    """研究智能体"""
    async def research(self, topic: str, sources: List[str]):
        materials = []
        
        # 多源检索
        if 'web' in sources:
            web_results = await self.web_search(topic)
            materials.extend(web_results)
        
        if 'knowledge_base' in sources:
            kb_results = await self.kb_search(topic)
            materials.extend(kb_results)
        
        if 'database' in sources:
            db_results = await self.db_query(topic)
            materials.extend(db_results)
        
        # 去重和排序
        return self.deduplicate_and_rank(materials)
```
